03What's added from 10x

Three components from the 10x Stage 4 (Close) and interview infrastructure — all currently "10x only" capabilities absent from the base 2x design.

Structured competency scorecards

The 10x Hiring Decision System (HDS) taxonomy — 7 evaluation categories with per-competency scoring and teachability coding. Claude generates question sets per competency; interviewer scores each live.

Where it lands: Immediately after the HR interview. Replaces the binary hire/no-hire gut call with a structured, auditable evaluation that feeds the go/no-go gate.

Reference checking

Structured outreach to candidate-provided references plus back-channel references where relevant. Results logged against the candidate record in Greenhouse before the offer is drafted.

Where it lands: First step of the Close stage, after scorecard passes. Required before offer goes into the approval chain.

Multi-level offer approval + AI commentary

Offer routes through a multi-level approval chain before extension. Claude drafts offer commentary contextualizing the package for the candidate. Offer letter generated and tracked.

Where it lands: Final step before offer extension. Consistent, approved offers — and a more professional candidate experience.

Multi-tenancy — architectural note

Multi-tenancy (~280 permissions, role-based access control, field-level audit) is a platform-layer property in 10x — not a process step. In the 2x context this translates to: design the Greenhouse + n8n integration with per-role and per-department permission boundaries from the start, so the system can serve multiple hiring managers without data bleed. Relevant at build time; not shown in the flowchart.
