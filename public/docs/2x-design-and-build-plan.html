Automate the wide top of the hiring funnel, concentrate human judgment on finalists, and condense the timeline — built on Greenhouse with n8n + Claude (already in use) and Criteria/CCAT (already licensed) as the validated assessment gate.

01The Design

Lightspeed hires today on Greenhouse used as a manual ATS — every resume read by hand, then a phone screen for social competence, then interviews. The bottleneck is the widest, most manual part of the funnel, the cycle is slow, and candidates often never hear back. The fix isn't a new ATS — it's automation built on tools they already have.

Today

  • Greenhouse as a system of record only
  • Every resume reviewed manually — the bottleneck
  • Phone screen gates social competence
  • Candidates often "never hear back"

Proposed

  • Greenhouse stays the backbone (don't replace)
  • n8n + Claude orchestrate & score — already in use
  • Criteria/CCAT auto-assessment gates 1000→200 — already licensed
  • Role-specific work-sample task, Claude-scored
  • Insights Discovery values assessment
  • Automated candidate comms at every stage

03Architecture — connect, don't replace

Greenhouse remains the source of truth. n8n orchestrates custom steps; Claude is the AI engine. Criteria connects natively. n8n, Claude, and CCAT are already in company use, and the Greenhouse API is available on all tiers — so this is wiring, not procurement.

05Straw man and risks

Proposed

  • Integrate, don't replace Greenhouse (n8n middleware)
  • Build on existing tools — n8n, Claude, CCAT already in use
  • Greenhouse API available on all tiers (no gate)
  • Sample task early (200→20) as the primary skills filter
  • Auto-reject cutoff <30 / 70th pct (provisional)
  • Insights Discovery assessment scored against values + job description

Risks & guardrails

  • Enabling the Criteria integration is the critical path
  • Prove the auto-reject is fair and accurate
  • Personally Identifiable Information (PII)/security review may gate go-live
  • Build on Harvest API v3 (v1/v2 retire 08-31-26)
  • Minimize rejection errors and the risk of losing strong candidates based on automation
